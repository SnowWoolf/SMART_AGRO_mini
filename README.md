## Первоначальная установка

Подключиться по SSH и выполнить 1 команду:
```bash
curl -fsSL https://raw.githubusercontent.com/SnowWoolf/SMART_AGRO_mini/main/install.sh | bash
```

## Обновление

Подключиться по SSH и выполнить 1 команду:
```bash
curl -fsSL https://raw.githubusercontent.com/SnowWoolf/SMART_AGRO_mini/main/update.sh | bash
```

Скрипт клонирует ветку `MAIN` во временную папку, сравнивает версии файлов и обновляет только те файлы, у которых версия в репозитории новее локальной.

Версия читается из первой строки файла. Если у файла в репозитории версии нет, файл пропускается. Если локального файла нет, но у файла в репозитории версия есть, файл будет добавлен. Если локальный файл есть, но у него нет версии, а у файла в репозитории версия есть, локальный файл будет обновлен.

Исключения не обновляются: `config.py`, `.env`, `*.db`, `main.html`, `main2.js`, а также содержимое `.git/*` и `camera_archive/*`.

Если у локального файла версия содержит постфикс `-noupd`, например `/* VERSION: 2.0.290426-noupd */`, такой файл не обновляется. В конце скрипт выводит список этих файлов с предупреждением: `ВНИМАНИЕ! Необходимо вручную проверить и обновить содержимое файлов:`.

Если обновления найдены, скрипт останавливает сервисы `agrosmart_web` и `agrosmart_sync`, делает полный backup файлов приложения, копирует обновляемые файлы, затем запускает сервисы обратно и проверяет, что они активны.

Если копирование файлов или запуск сервисов завершается ошибкой, скрипт откатывает файлы из backup, пытается снова запустить сервисы и завершает работу с ошибкой.

В конце успешного обновления выводится список обновленных файлов и, если есть, список файлов `-noupd` для ручной проверки.



## Статус служб
```bash
systemctl status agrosmart_web
```
```bash
systemctl status agrosmart_sync
```
